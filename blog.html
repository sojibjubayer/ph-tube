The scope of var, let, and const

The keywords var, let, and const allow us to declare variables. A variable can be declared in javascript using the venerable Var variable method. We utilize the let and const variables in the current javascript, which was included in the ES2015(ES6) update; it is now used more commonly in modern javascript than the var variable.

scope:Scope basically refers to the area in which these variables can be used. Var declarations can be function/locally or globally scoped.

scope of var

When a var variable is declared outside of a function, the scope is global. This means that any variable declared outside of a function block using the var keyword is accessible throughout the entire window. When a variable is declared inside a function, it has function scope. This indicates that it is accessible and limited to that function.

Hoisting of var: Before code is executed, variables and function declarations are hoisted to the top of their scope in JavaScript. As a result, var variables are initialized with the value undefined and raised to the top of their scope.

scope of let

Let is now the preferred syntax for declaring variables.A variable declared in a block with let can only be used in that block. A block is a section of code that is bounded by. Curly braces house a block. A block is anything contained in curly brackets. Let is a superior option to var due to this feature. If you've used a name for a variable before, you don't need to worry while using let because a variable only exists within its scope.

scope of const

Constant values are kept for variables specified using the const keyword. The scope of const declarations is blocked. Const declarations, like let declarations are only accessible within the block in which they were made. It is impossible to modify or re-declare const. This indicates that within the scope of a variable declared with const, its value will not change. It cannot be changed or declared again.

The use cases of null and undefined

undefined

undefined means a variable has been declared but has not yet been assigned a value. undefined is a type by itself (undefined). Unassigned variables are initialized by JavaScript with a default value of undefined.

A variable that has not been assigned a value is of type undefined. A method or statement also returns undefined if the variable that is being evaluated does not have an assigned value. A function returns undefined if a value was not returned.

While you can use undefined as an identifier (variable name) in any scope other than the global scope (because undefined is not a reserved word), doing so is a very bad idea that will make your code difficult to maintain and debug.

You can use undefined and the strict equality and inequality operators to determine whether a variable has a value.

null

On the other hand, null is an object. It can be assigned to a variable as a representation of no value. JavaScript never sets a value to null. That must be done programmatically.

A null value in JavaScript is used for referring absence of any object value and if any function or variable returns null, then we can infer that the object could not be created.

When used in conditional logic both undefined and null will return false.

null

1.Intentional Absence of Value: Developers can assign null to a variable or property to indicate that there is no value or that the value is intentionally missing or unknown.

let myVariable = null;

In this case, myVariable is explicitly set to null to signify that it doesn't have a meaningful value.

2.Function Return Values: Functions may return null to indicate that they did not produce a valid result or that an error occurred.

// ... logic to find target

}

undefined

1.Default Function Parameters: When a function is called with fewer arguments than declared parameters, the missing parameters are automatically set to undefined.

function greet(name, age) {
console.log(name); // If name is not provided, it's undefined
console.log(age); // If age is not provided, it's also undefined
}

2.Variable Declaration: Variables declared but not initialized are by default set to undefined.

let myVariable;
console.log(myVariable); // Outputs undefined

3.Object Properties: When you try to access an object property that doesn't exist, it returns undefined.

const person = { name: 'John' };
console.log(person.age); // Outputs undefined

4.Function Return Values: If a function doesn't explicitly return a value, it returns undefined by default.

function noReturnValue() {
// No return statement, so it returns undefined
}

5.Checking for Undefined: You can check if a variable or property is undefined to handle cases where a value might not exist or hasn't been assigned yet.

if (typeof someVariable === 'undefined') {
// Handle the case when someVariable is undefined
}

REST API

An API for a website is code that allows two software programs to communicate with each other.

A RESTful API -- also referred to as a RESTful web service or REST API -- is based on representational state transfer (REST), which is an architectural style and approach to communications often used in web services development.

A RESTful API is an architectural style for an application program interface (API) that uses HTTP requests to access and use data. That data can be used to GET, PUT, POST and DELETE data types, which refers to the reading, updating, creating and deleting of operations concerning resources.

REST technology is generally preferred over other similar technologies. This tends to be the case because REST uses less bandwidth, making it more suitable for efficient internet usage. RESTful APIs can also be built with programming languages such as JavaScript or Python.

How RESTful APIs work

A RESTful API uses commands to obtain resources. The state of a resource at any given timestamp is called a resource representation. A RESTful API uses existing HTTP methodologies defined by the RFC 2616 protocol, such as:

  • GET to retrieve a resource;
  • PUT to change the state of or update a resource, which can be an object, file or block;
  • POST to create that resource; and
  • DELETE to remove it.
